# VoteTerminal ‚Äî Self-Host Guide

Run your own private voting platform in under 15 minutes.

---

## What You Need

- A Linux server (Ubuntu 20.04+ or Debian 11+) with root access
- A domain or subdomain pointing to your server
- An email provider account (Resend is free for up to 3,000 emails/month)

---

## Quick Install

SSH into your server, upload the VoteTerminal package, then run:

```bash
chmod +x install.sh
sudo ./install.sh
```

The installer will ask you a few questions (domain, org name, admin email, password, email provider) and handle everything else automatically ‚Äî Node.js, pm2, Apache, SSL, and your `.env` file.

## Branding Your Installation

After install, you can customise the look of your ballot portal in two ways:

### Option 1 ‚Äî Admin Panel (easiest)

1. Log in at `https://yourdomain.com/admin`
2. Go to **‚öôÔ∏è Settings**
3. Update your organisation name, tagline, and logo URL

Your logo should be a publicly accessible image URL (upload it to your website, Imgur, or any CDN). For best results use a wide, horizontal image (around 400√ó100 px) with a transparent background on a dark background.

### Option 2 ‚Äî Environment variables (persists across restarts)

Edit your `.env` file on the server:

```bash
nano /var/www/voterterm/backend/.env
```

Add or update:

```
ORG_NAME=Gwinnett Democratic Party
ORG_TAGLINE=Official Ballot
ORG_LOGO_URL=https://yoursite.com/your-logo.png
ORG_BANNER_COLOR=#003087
```

Then restart:

```bash
pm2 restart voterterminal
```

## Email Setup

VoteTerminal needs email to send voter confirmation codes and ballot receipts.

### Resend (recommended ‚Äî free tier: 3,000 emails/month)

1. Sign up at [resend.com](https://resend.com)
2. Add your domain and verify DNS (SPF + DKIM records)
3. Create an API key
4. In your `.env`:
   ```
   EMAIL_PROVIDER=resend
   RESEND_API_KEY=re_your_key_here
   EMAIL_FROM=elections@yourdomain.com
   ```

### Other providers

| Provider | `.env` settings |
|----------|----------------|
| Gmail SMTP | `EMAIL_PROVIDER=smtp`, `SMTP_HOST=smtp.gmail.com`, `SMTP_PORT=587`, `SMTP_USER=you@gmail.com`, `SMTP_PASS=your-app-password` |
| Brevo | `EMAIL_PROVIDER=smtp`, `SMTP_HOST=smtp-relay.brevo.com`, `SMTP_PORT=587` |
| Mailgun | `EMAIL_PROVIDER=smtp`, `SMTP_HOST=smtp.mailgun.org`, `SMTP_PORT=587` |
| SendGrid | `EMAIL_PROVIDER=sendgrid`, `SENDGRID_API_KEY=SG.xxx` |
| No email (testing) | `EMAIL_PROVIDER=console` (emails print to logs) |

## Managing Your Installation

```bash
# View logs
pm2 logs voterterminal

# Restart after config changes
pm2 restart voterterminal

# Update to a new version
# 1. Upload new voting-app-server.js and email-service.js
# 2. pm2 restart voterterminal
# 3. Upload new React build/ folder (no restart needed)

# Check Apache
sudo apache2ctl configtest
sudo systemctl reload apache2
```

## Before a Live Election

- [ ] `NODE_ENV=production` in `.env`
- [ ] Strong `JWT_SECRET` (auto-generated by installer)
- [ ] Strong `ADMIN_PASSWORD` ‚Äî change it at `/admin` ‚Üí Settings
- [ ] Email provider configured and tested
- [ ] SSL certificate active (`https://` loads without browser warning)
- [ ] Test vote submitted and confirmation email received
- [ ] `pm2 startup` configured (installer does this automatically)

## Docker Install (alternative)

If you prefer Docker over a direct install:

```bash
# 1. Copy .env.example to .env and fill in your values
cp .env.example .env

# 2. Build and start
docker compose up -d

# 3. View logs
docker compose logs -f app
```

See `docker-compose.yml` and `docker/nginx.conf` for configuration details.

## Troubleshooting

**403 Forbidden** ‚Äî Apache DocumentRoot mismatch. Check:
```bash
sudo apachectl -S
sudo tail -20 /var/log/apache2/error.log
```

**API returning 503** ‚Äî Node.js isn't running. Check:
```bash
pm2 status
pm2 logs voterterminal --lines 30
```

**Emails not sending** ‚Äî Test with:
```bash
curl -X POST https://yourdomain.com/api/voter/register \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","electionId":"test"}'
```
Check `pm2 logs voterterminal` for email output.

**SSL renewal** ‚Äî Handled automatically. Test with:
```bash
sudo certbot renew --dry-run
```
